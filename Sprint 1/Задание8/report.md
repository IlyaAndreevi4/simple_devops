# Уровень логирования

---

## Проблема

При создании контейнера без явного указания `LOG_LEVEL` генерируются избыточно большие логи.

По умолчанию переменная задана на уровне `DEBUG` — как в `Dockerfile`:
```dockerfile
ENV LOG_LEVEL=DEBUG
```
так и в самом приложении:
```python
LOG_LEVEL = os.getenv('LOG_LEVEL', 'DEBUG').upper()
```

Уровень `DEBUG` даёт максимально подробный вывод — он уместен при отладке или поиске проблем, но не в рабочей среде.

---

## Решение

Переопределить `LOG_LEVEL` при запуске контейнера:

```bash
docker run -d -p 8080:8080 --name logtest -e LOG_LEVEL=INFO log-app
```

---

## Допустимые значения

| `LOG_LEVEL` | Описание                        |
|-------------|---------------------------------|
| `DEBUG`     | Максимально подробный вывод     |
| `INFO`      | Стандартный рабочий режим       |
| `PROD`      | Продакшн-режим                  |

> ⚠️ Любое другое значение (`WARNING`, `ERROR` и т.д.) отключит логирование полностью.
